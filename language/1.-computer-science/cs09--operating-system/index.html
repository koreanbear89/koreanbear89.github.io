1. Central Processing Unit

  • CPU / Processor : 보통 CPU 와 Processor 는 같은 의미로 사용됨

  • Core :

    • CPU를 내부에서 물리적으로 분할
    • 과거 1CPU = 1Core 인 시절에는 의미상 구분이 필요 없었으나
    • 최근에는 CPU 하나당 2,4개의 코어로 구성되어 1-Core CPU 대비 물리적으로 N개의 일을 동시에 처리할 수 있게됨

  • Thread

    • 쓰레드는 코어를 다시한번 논리적으로 구별한 것으로

    • 최근에는 하이퍼쓰레딩으로 1코어당 2쓰레드가 일반적이며 이 경우 코어수가 실제 코어수의 두배로 인식

    • 실제로 스레드지연시간을 획기적으로 줄일 수 있기에 체감이 큰 편 (밥을 한손으로 먹는것과 양손으로 먹는것의 차이)

    • ex) 두가지 작업 ‘0000’ 과 ‘1111’ 이 있다면 기존의 기술은 ‘00001111’ 의 방식으로 처리하는 반면 하이퍼스레딩은 ‘01010101’로 처리

      # 물리 CPU 개수
grep "physical id" /proc/cpuinfo | sort -u | wc -l
>> 2 

# cpu당 물리코어의 개수 
grep 'cpu cores' /proc/cpuinfo | tail -1
>> 14  # 가상코어의 개수는 28개

# 전체 시스템의 가상 코어 개수 
grep -c processor /proc/cpuinfo    
>> 56

  • Thread (OS)

    • 운영체제(OS)에서도 스레드라는 단어가 사용되는데
    • 이때 스레드의 기준은 하나의 프로세스(=어떤 프로그램)을 나눠서 처리할때의 단위입니다.
    • 프로세스도 cpu와 마찬가지로 프로세스 하나당 기본적으로 하나의 메인 스레드가 존재하며 다수의 스레드로 수행되는것을 멀티스레딩이라고 칭합니다. 요즘 흔히보이는 프로그램들은 거의다 멀티스레딩구조라고 봐도 무방합니다.
