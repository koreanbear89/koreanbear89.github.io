Recommendation System

  2021-01-14   6. Recommendation System  Comments

Introduction

  • Recommender system : is a subclass of information filtering sys that seeks to predict the “rating” or “preference” a user would give to an item.
  1. Content Based Filtering : 사용자가 특정 아이템을 선호하는 경우 그 아이템과 비슷한 콘텐츠를 가진 다른 아이템을 추천해주는 방식 (e.g. 마블영화를 본 사용자에게 다른 마블영화를 추천)
  2. Collaborative Filtering : 다수의 사용자 집단의 행동을 분석하고 이와 유사한 행동패턴의 사용자에게 같은 제품을 추천 (20대 게임을 좋아하는 남성들은 이런 컨텐츠를 즐겨찾아본다.)
    • User-based : 유저의 행위를 분석하여 유저간 유사도를 측정, 유사도가 높은 다른 유저들의 선호 아이템을 바탕으로 추천
    • Item-based : user based 방법은 선호도 파악이 어려운 새로운 유저와 선호도가 분산되는 오래된 유저에 대한 추천품질이 하락 => 따라서 제품간의 유사도를 측정하여 어떤 아이템을 선호하면 유사한 다른 아이템을 추천
    • Cold-Start : 분석에 사용할 데이터가 부족한 서비스 초기에는 추천결과가 부정확한 문제



Knowledge Graph (KG)

  • The knowledge graph (KG) represents a collection of interlinked descriptions of entities – real-world objects and events, or abstract concepts (e.g., documents) –

  • Key Characteristics

    • Database : because the data can be explored via structured queries
    • Graph : because they can be analyzed as any other network data structure
    • Knowledge base : because they bear formal semantics, which can be used to interpret the data and infer new facts.

Tools

  • Airflow : 는 Python 코드로 워크플로우(workflow)를 작성하고, 스케쥴링, 모니터링 하는 플랫폼입니다. Airflow를 통해서 데이터엔지니어링의 ETL 작업을 자동화하고, DAG(Directed Acyclic Graph) 형태의 워크플로우 작성이 가능합니다. 이를 통해 더 정교한 dependency를 가진 파이프라인을 설정할 수 있습니다. 또한 AWS, GCP 모두 Airflow managed service를 제공할 정도로 전세계 데이터팀들에게 널리 사용되고 있으며 그만큼 넓은 커뮤니티를 형성하고 있습니다. 참고
