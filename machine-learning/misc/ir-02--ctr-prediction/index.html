IR #2 | CTR Prediction

  2025-02-01   8. Information Retrieval  Comments

Introduction

  • Problem Definition

    • aims to predict the probability of a user clicking on an ad or an item (recommendation sys.)

      • Feature Interaction Modeling

      • Sequential Behavior Modeling

      • Integrate feature interaction and sequential recommendation Model

    • Challenges

      • the input features (e.g., the user id, user age, item id, item category) are usually sparse and high-dimensional

      • an effective prediction relies on high-order combinatorial features

        • Gender=Male, Age=10, ProductCategory=VideoGame

  • Overview

    • Feature Interaction : combine features that describe the current interaction and generate high-order feature combinations relevant to the prediction task.

      • (1) Factorization Machine : project sparse features onto low dimensional dense vectors and learn feature interactions from vector inner products

        • Factorization Machines

          • map each feature to latent space, get interaction using dot-product

- (2) Deep Neural Networks : Two Stream Architectures (architecture variants : stack, parallel, additional layers, etc.)

1. Factorization Machines (ICDM'10)

  • Introduction :

    • introduce Factorization Machines (FM) which are a new model class that combines the advantages of Support Vector Machines (SVM) with factorization models.

    • In contrast to SVMs, FMs model all interactions between variables using factorized parameters. Thus they are able to estimate interactions even in problems with huge sparsity (like recommender systems) where SVMs fail.


2. [Deep & Cross Network for Ad Click Predictions (Google, ADKDD'17)]([1708.05123] Deep & Cross Network for Ad Click Predictions)

  • Introduction : Authors aim to avoid task-specific feature engineering by introducing a novel neural network structure – a cross network – that explicitly applies feature crossing in an automatic fashion.
  • Methods : Deep Cross Network
    • Embedding and Stacking Layer : transform sparse (one-hot) vectors into dense vectors (embedding vectors) and stack the embedding vectors along with the normalized dense vectors
      • $x_0 = [ x^T_{emb1}, … , x^T_{embK} , x^T_{dense} ]$
    • Cross Network : apply explicit feature crossing in an efficient way
      • $x_1 = x_0x^T_0 w_{0} +b_{0} + x_0$
    • Deep Network : a simple fully connected neural net
    • Combination Layer : concat the outputs from two nets and feed it into a standard logit layer
      • $p = sigmoid(W_{logit}x_{stack} +b_{logit})$

3. AutoInt: Automatic Feature Interaction Learning via Self-Attentive Neural Networks

  • Introduction : propose a novel approach based on self-attentive neural network, which can automatically learn high-order feature interactions
  • Methods : Automatic Feature Interaction Learning (AutoInt)
    • Input Layer : concatenation of all fields $x_i$
      • $x_i$ is a scalar value if the i-th field is numerical
      • $x_i$ is a one-hot vector if the i-th field is categorical
    • Embedding Layer : represent sparse and high-dim features into low-dimensional spaces
      • the output of the embedding layer would be a concatenation of multiple embedding vectors
    • Interaction Layer : extend multi-head self-attention to model the correlations between different feature fields.
    • Output Layer : we simply concatenate output of interaction layer and then apply a non-linear projection as follows :
      • $\hat{y}= σ(w^T(e_1 ⊕ e_2 ⊕···⊕e_M )+ b)$

4. FinalMLP: An Enhanced Two-Stream MLP Model for CTR Prediction (AAAI'23)

  • Introduction

    • well-tuned two-stream MLP model that simply combines two MLPs can even achieve surprisingly good performance

    • We propose an enhanced two-stream MLP model, namely FinalMLP, which integrates feature gating and interaction aggregation layers on top of two MLP module networks

  • Methods : feature gating and interaction aggregation layers that can be easily plugged to make an enhanced two-stream ML

    • Stream Specific Feature Selection : reweighting feature inputs differently for each stream.

    • Stream-Level Interaction Aggregation :

      • Bilinear Fusion

        • existing work mostly employs summation or concatenation as the fusion layer.

        • we propose a bilinear interaction aggregation layer to fuse the stream outputs with stream-level feature interaction.

        • The bilinear term models the second-order interactions between o1 and o2.

          • $ y= σ(b+ w^T_1 o_1 + w^T_2 o_2 + o^T_1 W_3o_2), $

      • Multi-Head Bilinear Fusion

        • perform the bilinear fusion in each subspace that pairs $o_{1j}$ and $o_{2j}$ as a group.

        • Then, we aggregate the subspace computation by sum pooling
