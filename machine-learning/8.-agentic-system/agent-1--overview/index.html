1. A Survey on Large Language Model based Autonomous Agents (2023)

  • Abstract : The paper surveys the emerging paradigm of LLM-based autonomous agents, which extend large language models beyond passive text generation to active systems capable of reasoning, planning, memory, tool use, and interaction with environments.
  • Introduction :
    • Definition : An LLM-based autonomous agent is a system where an LLM acts as the central controller, orchestrating perception, reasoning, memory, and actions to achieve goals with minimal human intervention.
    • Motivation : Traditional AI systems are task-specific and rigid, while LLMs show strong general reasoning and language capabilities. Combining LLMs with external tools and environments leads to autonomous agents that can: (1) Understand goals, (2)Plan multi-step actions, (3) Interact with tools and environments, (4) Adapt through feedback and memory
  • Agent Construction (Agent Architecture Design)
    • Reasoning : LLM acts as the central reasoning and decision-making engine
    • Profile : The purpose of the profiling module is to identify the role of the agent. This is basically prompt engineering as agent identity which defines - goals, constraints, personality, system prompt.
      • Handcrafting Method : agent profiles are manually specified.
      • LLM Generation Method : agent profiles are automatically generated based on LLMs.
      • Dataset Alignment Method: in this method, the agent profiles are obtained from real-world datasets.
    • Memory : The memory module stores information perceived from the environment and leverages the recoreded memories to facilitate future actions. This section provides a comprehensive overview of the memory module focusing on its :
      • Structures : short-term memory is analogous to the input information within the context window constrained by the transformer architecture. Longterm memory resembles the external vector storage that agents can rapidly query and retrieve from as needed.
        • Unified Memory (short-term memory in prompt) : only simulates the human short-term memory, which is usually realized by in-context learning, and the memory information is directly written into the prompts.
        • Hybrid Memory (short & long term memory) : The short-term memory temporarily buffers recent perceptions, while long-term memory consolidates important information over time.
      • Formats : Natural Languages, Embeddings, Databases
      • Operations : Memory Reading, Memory Writing, Memory Reflections
    • Planning : the memory and planning modules place the agent into a dynamic environment, enabling it to recall past behaviors and plan future actions.
      • Planning without Feedback:
        • Single-path Reasoning : the final task is decomposed into several intermediate steps
        • Multi-path Reasoning : the reasoning steps for generating the final plans are organized into a tree-like structure.
        • External Planner : effectively generating plans for domain-specific problems remains highly challenging. To address this challenge, researchers turn to external planners.
      • Planning with Feedback:
        • Environmental Feedback : This feedback is obtained from the objective world or virtual environment.
        • Human Feedback : directly interacting with humans is also a very intuitive strategy to enhance the agent planning capability.
        • Model Feedback : utilization of internal feedback from the agents themselves.
    • Action : The action module is responsible for translating the agent’s decisions into specific outputs.
      • Action Goal
        • Task Completion : The actions are aimed at accomplishing specific tasks
        • Communication : with the other agents or real humans for sharing information
        • Environment Exploration : explore unfamiliar environments to expand its perception
      • Action Production : Different from ordinary LLMs, where the model input and output are directly associated, the agent may take actions via different strategies and sources.
        • Action via Memory Recollection : the action is generated by extracting information from the agent memory according to the current task.
        • Action via Plan Following : In this strategy, the agent takes actions following its pre-generated plans.
      • Action Space : refers to the set of possible actions that can be performed by the agent.
        • external tools : APIs, DB & KB, External Models
        • internal knowledge of the LLMs : Planning, Conversation, Common Sense Understanding
      • Action Impact : refers to the consequences of an agent’s actions.
        • Changing Environments.
        • Altering Internal States
        • Triggering New Actions.
  • Agent Applications
    • This section provides a succinct summary of previous studies, categorizing them according to their applications in three distinct areas: social science, natural science, and engineering.
  • Agent Evaluation
    • Subjective methods : Human annotation, Turing Test,
    • Objective methods : Task success metrics (acc, err rate), Human similarity metrics, Efficiency metrics

2.The Rise and Potential of Large Language Model Based Agents

  • Introduction :
    • LLMs are evolving from language models into autonomous agents capable of reasoning, planning, and acting.
    • LLM-based agents integrate models, memory, tools, and interaction to solve complex tasks.
    • Their rapid development highlights both significant potential and critical challenges in reliability and scalability.
  • Construction of LLM based Agents
    • Brain : Natural Language Interaction, Knowledge, Memory, Reasoning and Plain
    • Perception : Textual, Visual, Auditory, Other input
    • Action : Textual Output, Tool Using, Embodied Action
  • Agents in Practice : Harnessing AI for good
    • Single-Agent Applications
      • Perform autonomous task execution using LLM reasoning and tool use.
      • Applied in task-oriented, creative, and long-term lifecycle scenarios.
      • Limited by reasoning reliability, environment understanding, and robustness.
    • Multi-Agent Systems
      • Multiple agents collaborate or compete to solve complex tasks.
      • Enable role specialization, distributed reasoning, and emergent behavior.
      • Face challenges in coordination, communication overhead, and alignment.
    • Human-Agent Systems
      • Humans guide agents through instructions or feedback loops.
      • Agents act as assistants or partners in decision-making and creativity.
      • Practical value emerges from effective human–agent interaction rather than full autonomy.
