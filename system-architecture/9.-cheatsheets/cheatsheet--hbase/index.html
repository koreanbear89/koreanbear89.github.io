Introduction

  • Hbase : Hadoop database, a distributed, scalable, big data store

  • Architecture / Components

    • Master Server (HMaster)

    • Region Server

    • Zookeeper

  • When shoud I use Hbase?

    1. Make sure you have enough data : more than hundreds of millions or billions of rows. If not using traditional RDBMS migh be a better choice

    2. Make sure you can live without all extra features that an RDBMS provides (e.g., typed columns, secondary indexes, transactions, advanced query languages, etc.)

    3. Make sure you have enough hardware Even HDFS does not do well with anything less than 5 data nodes + NameNode (default HDFS block replication 3)

4. Spark-Hbase integration

  • Setup

    • import org.apache.hadoop.hbase.HBaseConfiguration

      • run spark with option --jars "${HBASE_HOME}/lib/*.jar"
      • hbase-site.xml should be made available to Spark, it can be copied to $SPARK_CONF_DIR (default is $SPARK_HOME/conf)

    • import org.apache.hadoop.hbase.spark.HBaseContext

      • need Apache Hbase Spark Connectors (download-jar | build)

        • If got error : Symbol term org.apache.yetus is missing from the classpath … `

          •  spark cannot load hbase jar => cp  $HBASE_HOME/lib/client-facing-thirdparty/audience-annotations-*.jar $SPARK_HOME/jars
import org.apache.hadoop.hbase.spark.HBaseContext
import org.apache.hadoop.hbase.HBaseConfiguration
val conf = new HBaseConfiguration()    
conf.set("hbase.zookeeper.quorum", "hostname1,hostname2...")  
conf.set("hbase.zookeeper.property.clientPort", "2181") 
new HBaseContext(spark.sparkContext, conf)
