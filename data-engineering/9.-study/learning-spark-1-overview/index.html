Learning Spark #1 | Introduction to Apache Spark

  2024-10-22   2. Backend  Comments

1.1 The Genesis of Spark

  • Google :

    • Traditional storage systems such as RDBMSs were not able to handle the scale at which Google wanted to build and search the internet’s indexed documents.

      • need for new approaches led to the creation of the Google File System (GFS), MapReduce (MR), and Bigtable.

  • Yahoo :

    • Hadoop Common, MapReduce, HDFS, and Apache Hadoop YARN.

      • intermediate computed result is written to the local disk for the subsequent stage of its operation

      • short for combining other workloads such as machine learning, streaming, or interactive SQL-like queries. => Apache Hive, Storm, Impala, Giraph, Dril.

  • Spark

    • A way to make Hadoop and MR simpler and faster? => Spark

1.2 What Is Apache Spark

  • Key Characteristics:

    • Speed

      • With all the intermediate results retained in memory and its limited disk I/O, this gives it a huge performance boost.

    • Ease of use :

      • High level structured data abstractions, such as DataFrames and Datasets

      • By providing a set of transformations and actions as operations, Spark offers a simple programming model that you can use to build big data applications in familiar languages.

    • Modularity

      • Spark SQL, Spark Structured Streaming, Spark MLlib, and GraphX

    • Extensibility

      • You can use Spark to read data stored in myriad sources—Hadoop, Apache Cassandra, Apache HBase, MongoDB, Apache Hive, RDBMSs, and more—and process it all in memory.

1.3 The Developer’s Experience

  • Data Science Tasks :

    • have to cleanse the data, explore it to discover patterns,

    • build models to predict or suggest outcomes.

  • Data Engineering Tasks

    • build and transform raw, dirty data into clean data
    • skills for building scalable data pipelines for a stated business use case

  • Popular Spark use cases

    • Processing in parallel large data sets distributed across a cluster

    • Performing ad hoc or interactive queries to explore and visualize data sets

    • Building, training, and evaluating machine learning models using MLlib

    • mplementing end-to-end data pipelines from myriad streams of data

    • Analyzing graph data sets and social networks


2.3 Understanding Spark Application Concepts

  • Spark Application : A user program built on Spark using its APIs. It consists of a driver program and executors on the cluster

  • SparkSession : An object that provides a point of entry to interact with underlying Spark functionaility and allows programming Spark with its APIs

  • Spark Jobs : A parallel computation consisting of multiple tasks that gets spawned in response to a Spark acion

    • The driver converts Spark Application into one or more Spark jobs

  • Spark Stages : Each job gets divided into smaller sets of tasks called stages that depend on each other

    • stages are created based on what operations can be performed serially or in parallel

  • Spark Tasks : A single u/user/video/search/collection/main/video.main.d01.20241205042049nit of work or execution that will be sent to a Spark executor.

    • Each task maps to a single core and works on a single partition of data, making the execution of tasks exceedingly parallel.

  • Spark Operations : can be classified into two types :

    • Transformations : transform a Spark DF into a new DF

      • Narrow Transformations : A single output partition can be computed from a single input partition

        • filter(), select()

      • Wide Transformations : Data from other partitions is read in, combined, and written to disk.

        • groupBy(), orderBy()

      • join() can ve wide or narrow depending on the partitioning of the input.

    • Actions : trigger the lazy evaluation of all recorded transformations.

      • show(), take(), count(), collect(), save()

    • Lazy Evaluation : All transformations are evaluated lazily, not computed immediately
